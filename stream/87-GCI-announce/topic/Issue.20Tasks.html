@all In case you don't know you are free to work on open issues in any of the Zulip repos. If there is no task created already for that issue we will create one for you once your PR gets merged :)

Rishi Gupta (Dec 12 2017 at 18:02):

As a quick tip, if you're going to start looking through the zulip repos:

  • Read zulip.readthedocs.io/en/latest/overview/contributing.html
  • Only claim 1 issue at a time
  • Only claim issues that have the good first issue or help wanted labels; if it doesn't have one of those labels it means the issue is waiting on something else and is not currently actionable.

Rishi Gupta (Dec 12 2017 at 19:44):

Actually, I just realized directly claiming tasks is against the GCI rules, since it would allow students to claim more than one task at a time (one in the GCI app, and one in the repo).

If there is an issue that you want to work on, and that we haven't already opened as a GCI task, ask a mentor to create it as a task. (This process might take a day or two.) Many of the appropriately-sized issues should already be opened as GCI tasks though, and we add more as we see them, so don't feel a need to look through the issue lists if you're finding good tasks within the GCI app.
